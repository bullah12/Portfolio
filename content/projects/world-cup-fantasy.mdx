## Key decisions

Service-role credentials remain on the Worker, while the browser receives only the permissions it needs. Result polling maps external team identities before upserting only changed matches, allowing Supabase Realtime to refresh the table without noisy writes.

The prediction model covers both group-stage scores and knockout-specific penalty winners. Locking and scoring stay configurable so the same core flow can handle different league rules without a rewrite.

## Architecture

The browser owns the fast prediction experience. Supabase provides Postgres, authentication and realtime updates. A Cloudflare Worker is the narrow automation boundary between the external scoreboard feed and the application database.
