## Key decisions

Filter state lives in the URL so the catalogue is shareable and browser navigation remains useful. The visual system treats type, spacing and motion as reusable primitives rather than page-specific decoration.

The complete shopping journey is deliberately marked as client-side. It is a product and design-system template, not a claim that payments, identity or order fulfilment are connected.

## Architecture

A static React application uses Context for local shopping state and React Router for catalogue and content journeys. It is deployable as a front-end template without a server dependency.
